El arranque seguro de un sistema comienza desde la BIOS o UEFI, encargada de reconocer el hardware y decidir qué dispositivo iniciará primero.

Una configuración correcta en este nivel evita accesos no autorizados y ataques físicos al equipo.

Nota: La BIOS (Basic Input/Output System) es el firmware tradicional presente en equipos más antiguos, mientras que UEFI (Unified Extensible Firmware Interface) es su sucesor moderno, ofreciendo interfaz gráfica, soporte para discos grandes (GPT) y mayor seguridad gracias a funciones como Secure Boot o TPM (Trusted Platform Module).

Riesgos comunes si no se realiza un bastionado adecuado:

  • Acceso físico no autorizado para modificar el orden de arranque.
  • Instalación de software malicioso antes del sistema operativo (ataques tipo bootkit o rootkit).
  • Clonado o extracción de datos desde medios externos.
  • Desactivación de medidas de seguridad como Secure Boot o TPM.

2.1. Configuración de la contraseña de usuario o Power-On

Objetivo:
 Evitar que el sistema arranque sin autenticación.

Pasos:

  1. Reinicia el equipo y accede a la BIOS/UEFI presionando la tecla correspondiente (por ejemplo, F2, DEL, o ESC según el fabricante).
  2. Entra al menú Security o Password Settings.
  3. Busca la opción Power-On Password o User Password.
  4. Introduce una contraseña segura y confírmala.
  5. Guarda los cambios y reinicia el sistema (F10 o "Save & Exit").

Resultado:
 Cada vez que se inicie el equipo, se pedirá la contraseña antes de cargar cualquier sistema operativo.

Consejo adicional: Utiliza una contraseña alfanumérica con al menos 8 caracteres. Evita usar contraseñas similares a la del sistema operativo. En algunos equipos, la BIOS puede bloquearse por varios intentos fallidos, por lo que conviene mantener un registro seguro de la clave.

2.2. Configuración de la contraseña de administrador

Objetivo:
 Proteger la configuración interna de la BIOS/UEFI.

Pasos:

  1. Accede nuevamente a la BIOS/UEFI.
  2. Ve a Security > Administrator Password o Supervisor Password.
  3. Crea una contraseña distinta de la del arranque.
  4. Guarda los cambios y reinicia.

Resultado:
 Solo quien conozca la contraseña de administrador podrá modificar la configuración de la BIOS/UEFI.

Importante: La contraseña de administrador no impide el arranque del equipo, pero bloquea cambios críticos como el orden de arranque, la activación/desactivación de Secure Boot o la gestión del TPM. En entornos empresariales, esta contraseña suele gestionarse de forma centralizada mediante políticas de seguridad.

2.3. Evitar el arranque desde dispositivos externos

Objetivo: Impedir que alguien acceda al sistema mediante un USB o DVD externo.

Pasos:

  1. Accede a la BIOS/UEFI.
  2. Entra en Boot Settings o Boot Options.
  3. Deshabilita las opciones de arranque desde:
    • USB
    • CD/DVD
    • Network Boot (PXE)
  4. Guarda los cambios y reinicia.

Resultado: El equipo solo podrá arrancar desde el disco interno configurado como principal.

Recomendación adicional: Si en algún momento se necesita arrancar desde un medio externo (por ejemplo, para mantenimiento o recuperación), es posible habilitar temporalmente el arranque USB solo bajo supervisión del administrador.

2.4. Establecer un orden de arranque seguro.

Objetivo:
 Asegurar que el sistema operativo legítimo sea el primero en ejecutarse.

Pasos:

  1. En la BIOS/UEFI, entra en el menú Boot Order o Boot Priority.
  2. Coloca el disco duro principal (HDD/SSD) en la primera posición.
  3. Deja los demás dispositivos deshabilitados o en segundo lugar.
  4. Guarda los cambios y reinicia.

Resultado:
 El sistema buscará el sistema operativo en el disco interno antes que en cualquier otro dispositivo.

Complemento técnico: En sistemas UEFI, se pueden definir Boot Entries específicas, lo que permite asociar directamente un archivo EFI firmado (por ejemplo, \EFI\Microsoft\Boot\bootmgfw.efi). Esto refuerza la integridad del arranque al impedir el uso de cargadores no firmados.

2.5. Otras opciones de seguridad (2 pt)

Recomendaciones adicionales:

  • Activar Secure Boot: Solo permite ejecutar software firmado digitalmente.
  • Deshabilitar Wake on LAN (WoL): Evita el encendido remoto no autorizado.
  • Proteger físicamente la BIOS: Sellar o bloquear la carcasa del equipo.
  • Cifrar discos: En caso de sustracción física, los datos permanecerán inaccesibles.

Un correcto bastionado de la BIOS/UEFI es la primera barrera de defensa en la seguridad de un sistema.
